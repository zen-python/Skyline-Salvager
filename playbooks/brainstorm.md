# Brainstorm Playbook

Use this to move from an idea or theme to a structured concept document.

## Inputs

- user interests
- genre, theme, or platform hints
- constraints on team size, scope, or timeline

## Process

1. Generate multiple concept seeds.
2. Compare them by core fantasy, audience, and differentiation.
3. Narrow to 2-3 candidates.
4. Define pillars, anti-pillars, and the core loop.
5. Identify the MVP and the biggest technical risk.
6. Write the concept to `design/gdd/game-concept.md` using `templates/game-concept.md`.

## Roles

- primary: `agents/leads/game-designer.md`
- consult: `agents/directors/creative-director.md`
- consult for feasibility: `agents/directors/technical-director.md`

## Stop Points

- concept selection
- pillar approval
- MVP and scope tier approval
- write approval
