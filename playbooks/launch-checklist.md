# Launch Checklist Playbook

Use this for full launch readiness across engineering, QA, content, store,
operations, and community.

## Inputs

- launch date or dry-run mode
- release checklist if one exists
- latest milestone
- platform targets
- operational and community constraints

## Process

1. Review build and code health.
2. Review content and localization readiness.
3. Review QA status and performance targets.
4. Review store, legal, and distribution readiness.
5. Review infrastructure, analytics, and monitoring readiness.
6. Review marketing, support, moderation, and on-call readiness.
7. Produce an overall ready, not ready, or conditional verdict.

## Recommended Roles

- primary: `agents/directors/producer.md`
- consult: `agents/directors/technical-director.md`
- consult: `agents/leads/qa-lead.md`
- consult when needed: release-manager equivalent process in the target project

## Output

- launch checklist
- blocking items
- conditional items
- sign-off list
