# Reverse Document Playbook

Use this to derive design, architecture, or concept documentation from existing
code or prototypes.

## Modes

- design
- architecture
- concept

## Process

1. Read the implementation or prototype.
2. Extract rules, systems, dependencies, and technical decisions.
3. Ask intent-focused questions where code cannot explain motivation.
4. Generate the corresponding document in the target artifact format.

## Recommended Roles

- design mode: `agents/leads/game-designer.md`
- architecture mode: `agents/directors/technical-director.md`
- concept mode: `agents/directors/creative-director.md`
