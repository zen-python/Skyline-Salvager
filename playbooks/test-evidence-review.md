# Test Evidence Review Playbook

Use this to evaluate the quality of tests and manual evidence, not just whether
files exist.

## Inputs

- story, sprint, or system scope
- test files
- evidence documents
- acceptance criteria

## Process

1. Locate the stories in scope.
2. Find the expected test and evidence files.
3. Review automated tests for assertion coverage, edge cases, and naming.
4. Review manual evidence for completeness and sign-off quality.
5. Produce an ADEQUATE, INCOMPLETE, or MISSING verdict per story.

## Recommended Roles

- primary: `agents/leads/qa-lead.md`
- consult: `agents/specialists/qa-tester.md`
