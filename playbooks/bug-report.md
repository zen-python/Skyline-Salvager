# Bug Report Playbook

Use this to create, analyze, verify, or close a structured bug report.

## Modes

- description
- analyze
- verify
- close

## Process

1. Parse the mode.
2. Gather reproduction context.
3. Identify likely systems and files.
4. Classify severity and priority.
5. Write or update the bug report in `production/qa/bugs/`.

## Recommended Roles

- primary: `agents/specialists/qa-tester.md`
- consult: `agents/leads/qa-lead.md`
- consult: `agents/leads/lead-programmer.md` for likely root cause analysis
