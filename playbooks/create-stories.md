# Create Stories Playbook

Use this to turn system design into executable engineering stories.

## Inputs

- GDD or technical design doc
- architecture decisions
- current sprint or milestone goal

## Process

1. Identify the smallest reviewable increments.
2. Separate implementation, content, integration, and test work.
3. For each story, define:
   - goal
   - affected files or systems
   - dependencies
   - acceptance criteria
   - test notes
4. Keep stories small enough to implement and review in one focused cycle.

## Roles

- primary: `agents/directors/producer.md`
- consult: `agents/leads/lead-programmer.md`
- consult: `agents/leads/qa-lead.md`

## Output

Write the stories into the target project's production planning area.
