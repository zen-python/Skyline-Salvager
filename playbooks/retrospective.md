# Retrospective Playbook

Use this after a sprint or milestone to analyze what happened and define
actionable improvements.

## Inputs

- sprint or milestone plan
- delivered work
- git history
- blockers and debt trends
- previous retrospectives if any

## Process

1. Load the target sprint or milestone.
2. Compare planned work to delivered work.
3. Identify carryover, scope change, and unplanned work.
4. Assess debt and blocker patterns.
5. Produce a retrospective with concrete changes for the next cycle.
